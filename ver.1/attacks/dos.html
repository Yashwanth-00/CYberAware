Denial-of-Service (DoS) Attacks

A Denial-of-Service (DoS) attack aims to make a system, service, or network unavailable to legitimate users by overwhelming it with excessive traffic or resource requests. The goal is not to steal data, but to disrupt availability.

Types of DoS Attacks

  • Volumetric Attacks: Flooding a target with massive amounts of traffic (e.g., UDP flood).
  • Protocol Attacks: Exploiting weaknesses in network protocols (e.g., SYN flood).
  • Application Layer Attacks: Targeting specific applications (e.g., HTTP flood).
  • Distributed DoS (DDoS): Using multiple compromised devices (botnet) to attack simultaneously.

How DoS Attacks Work

  • Attacker generates excessive traffic or malformed requests.
  • Server resources (CPU, memory, bandwidth) become exhausted.
  • Legitimate users are unable to access the service.

In DDoS attacks, thousands of compromised machines (botnets) are used to amplify traffic, making mitigation more difficult.

Impact on Security (CIA Model)

  • Availability: Service becomes inaccessible to users.
  • Integrity: In some cases, logs and monitoring systems may be disrupted.
  • Confidentiality: Usually not directly affected, but may be targeted during downtime.

Detection Indicators

  • Sudden spikes in traffic volume.
  • Unusual network patterns or repeated connection requests.
  • High server CPU or memory utilization.
  • Increased latency or complete service failure.

Mitigation & Defense Strategies

  • Deploy Web Application Firewalls (WAF).
  • Use Content Delivery Networks (CDN) for traffic distribution.
  • Implement rate limiting and traffic filtering.
  • Use DDoS protection services (e.g., cloud-based scrubbing centers).
  • Configure firewalls and intrusion detection systems (IDS).
