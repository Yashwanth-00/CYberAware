Man-in-the-Middle (MitM) Attacks

A Man-in-the-Middle (MitM) attack occurs when an attacker secretly intercepts and potentially modifies communication between two parties without their knowledge. Instead of directly attacking a system, the attacker positions themselves between the sender and receiver.

Common MitM Techniques

  • Packet Sniffing: Capturing unencrypted network traffic.
  • Session Hijacking: Stealing session cookies to impersonate users.
  • ARP Spoofing: Redirecting traffic within local networks.
  • DNS Spoofing: Redirecting users to malicious websites.
  • SSL Stripping: Downgrading HTTPS connections to HTTP.

How MitM Attacks Work

  • User connects to a network (often public Wi-Fi).
  • Attacker intercepts traffic between user and server.
  • Data is monitored, modified, or redirected.
  • Victim remains unaware of the interception.

If communication is not encrypted (HTTP instead of HTTPS), sensitive information like passwords and session tokens can be exposed.

Impact on Security (CIA Model)

  • Confidentiality: Exposure of sensitive communication data.
  • Integrity: Manipulation of messages or transactions.
  • Availability: Possible disruption or redirection of traffic.

Detection Indicators

  • Certificate warnings in browsers.
  • Unexpected HTTPS to HTTP redirection.
  • Unusual DNS responses.
  • Session expirations or forced re-authentication.

Mitigation & Defense Strategies

  • Always use HTTPS with valid TLS certificates.
  • Enable HTTP Strict Transport Security (HSTS).
  • Avoid unsecured public Wi-Fi or use a VPN.
  • Implement certificate pinning in applications.
  • Use secure DNS services.
